# Regulate KR

> 대한민국 행정규칙(고시)을 Git으로 관리합니다.

[![규정](https://img.shields.io/badge/고시-10,189개-blue)](kr/) [![파일](https://img.shields.io/badge/파일-10,631개-green)](kr/) [![부처](https://img.shields.io/badge/전_부처-망라-orange)](kr/) [![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

대한민국 현행 고시 **10,189개**를 Markdown + YAML frontmatter로 변환하여 Git 저장소에서 관리합니다. 각 개정은 실제 시행일자를 가진 **Git commit**으로 추적됩니다.

[legalize-kr](https://github.com/legalize-kr/legalize-kr)이 법률 6,907개를, **regulate-kr**이 행정규칙(고시) 10,189개를 담당합니다.

## 왜 필요한가?

법률은 뼈대이고, **행정규칙(고시)이 실무**입니다.

금융 준법감시, 핀테크 인허가, 환경 규제, 의약품 허가, AI 규제 — 실제 기준은 법률이 아니라 각 부처의 **고시**에 있습니다.

```
자본시장법 제117조 (건전성 기준)
  └─ 금융투자업규정 제3조 (세부 기준)      ← 여기에 있음
       └─ [별표 2] 영업용순자본 산정기준   ← 진짜 기준은 여기
```

legalize-kr이 법률 6,907개를 Git으로 올렸다면, regulate-kr은 그 아래 계층인 **고시 10,074개**를 올렸습니다.

## 빠른 시작

```bash
git clone https://github.com/wellsa-ai/regulate-kr.git
cd regulate-kr

# 금융소비자보호 감독규정 보기
cat kr/금융소비자보호에관한감독규정/고시.md

# 개정 이력 보기
git log --format="%ai %s" -- kr/전자금융감독규정/

# 클라우드 보안 평가 기준 (별표)
cat kr/전자금융감독규정/별표2의2.md

# 전체 규정에서 "개인정보" 검색
grep -r "개인정보" kr/

# 두 감독규정 비교
diff kr/은행업감독규정/고시.md kr/상호저축은행업감독규정/고시.md
```

## 구조

```
kr/{규정명}/
  고시.md           # 감독규정 본문 (조문)
  별표1.md          # 별표 (세부 기준, 산정표 등)
  별표2.md
  별지1.md          # 별지 (신청서, 보고서 서식)
  ...
```

## 메타데이터 (YAML Frontmatter)

각 Markdown 파일에 YAML frontmatter로 메타데이터가 포함됩니다.

```yaml
---
제목: 전자금융감독규정
행정규칙구분: 고시
소관부처: 금융위원회
시행일자: "2026-02-13"
발령번호: "2026-7"
admRulSeq: "2100000274812"
출처: https://www.law.go.kr/행정규칙/전자금융감독규정
---
```

## 활용 사례

- **준법감시(Compliance)**: 감독규정 변경 시 `git diff`로 변경 조문 파악 → Gap 분석
- **핀테크 인허가**: 전자금융감독규정 별표로 보안 평가 기준 확인
- **RegTech**: 규정 텍스트를 RAG/벡터DB에 넣어 자연어 검색
- **법률 AI**: 법률(legalize-kr) + 감독규정(regulate-kr) 통합 학습 데이터
